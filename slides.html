The model internally knows it should call a tool, but the cognition vector and the execution vector become nearly orthogonal in the final layer.

Result: the model confidently gives wrong raw output instead of using available tools.

2 / 8

Three Root Causes

1

🧠 Attention Scarcity

Transformer n² attention diluted across long contexts. Models lose focus on tool descriptions buried in 10K+ token prompts. Every new token consumes finite "attention budget."

2

🧩 Representation Collapse

Cognition ("I need tool X") and execution ("call tool X") are encoded in nearly orthogonal subspaces at readout. The signal is there — it just can't reach the action layer.

3

🛠️ Tool Description Bloat

Overlapping descriptions create ambiguous selection signals. Models pick the wrong tool or skip tool use entirely. Tool descriptions are the #1 signal, not system prompts.

3 / 8

The Three-Tier Solution

1

Exhaustive Tool Review

Before concluding, scan ALL available tools. Classify by category. Force a second check: "What tool haven't I tried?"

Guards: Knowing-Doing Gap
2

Cross-Verify Search

≥2 independent sources. Official > journalism > social. Never trust single-source. Search before speaking, not after being corrected.

Guards: Single-source bias
3

Systematic Fact-Check

SIFT → CoVe+FIRE → FABLE → Truth Sandwich. Four-round progressive verification. Each round reduces one dimension.

Guards: Factual errors

Each tier catches what the previous one missed. Defense in depth.
